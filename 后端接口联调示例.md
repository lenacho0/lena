# 后端接口联调示例（前端 MVP 契约冻结版）

本文档用于前端 MVP 联调，冻结以下关键接口契约：

- `GET /api/batches/:id`（轮询任务状态主入口）
- `PATCH /api/batches/:id`
- `POST /api/batches/:id/reference-videos/select`
- `POST /api/batches/:id/script-generation-tasks`

## 1. 本地启动

```bash
npm run dev
```

默认地址：

```text
http://localhost:3000
```

seed 后可直接联调的固定 ID：

```text
batchId=00000000-0000-0000-0000-000000000001
referenceVideoId=00000000-0000-0000-0000-000000000101
```

## 2. 推荐联调顺序

1. `GET /api/batches/:id`（确认批次存在）
2. `PATCH /api/batches/:id`（补产品资料与目标脚本数）
3. `POST /api/batches/:id/reference-videos/select`（选中参考视频）
4. `POST /api/batches/:id/script-generation-tasks`（创建脚本生成任务）
5. 轮询 `GET /api/batches/:id` 的 `data.tasks` 与 `data.scripts`

## 3. POST /api/batches/:id/script-generation-tasks（冻结契约）

用途：

- 创建脚本生成任务（LLM MVP）
- 批次状态切换到 `GENERATING_SCRIPTS`
- 支持同批次活跃任务去重

请求示例：

```bash
curl -X POST "http://localhost:3000/api/batches/00000000-0000-0000-0000-000000000001/script-generation-tasks" \
  -H "Content-Type: application/json" \
  -d '{
    "desiredCount": 10,
    "priority": 120,
    "maxRetries": 3,
    "triggerSource": "ui_manual",
    "styleMix": {
      "stable_conversion": 0.4,
      "strong_hook": 0.4,
      "atmosphere_seeding": 0.2
    },
    "requestPayload": {
      "promptTemplateVersion": "v1",
      "operator": "frontend-debug"
    }
  }'
```

`202` 成功响应核心字段：

```json
{
  "data": {
    "task": {
      "id": "uuid",
      "batchId": "uuid",
      "taskType": "SCRIPT_GENERATION",
      "status": "QUEUED",
      "priority": 120,
      "requestPayload": {
        "desiredCount": 10,
        "triggerSource": "ui_manual",
        "executionMode": "llm_mvp"
      },
      "resultPayload": {
        "generationMethod": "llm_mvp",
        "workerStatus": "not_started"
      },
      "retryCount": 0,
      "maxRetries": 3
    },
    "batch": {
      "id": "uuid",
      "status": "GENERATING_SCRIPTS",
      "scriptTargetCount": 10
    },
    "executionMode": "llm_mvp",
    "deduplicated": false
  }
}
```

去重行为：

- 同 payload + 同批次存在活跃脚本任务：`202` 且 `deduplicated=true`
- 不同 payload + 同批次存在活跃脚本任务：`409`

常见失败：

- `400`：批次归档、缺产品资料、未选参考视频、字段类型非法
- `404`：`batchId` 不存在
- `409`：同批次有活跃脚本任务且 payload 不同

## 4. GET /api/batches/:id（轮询契约）

前端轮询关注：

- `data.status`
- `data.scriptGeneratedCount`
- `data.scripts[]`
- `data.tasks[]`（按 `createdAt desc`）

`data.tasks[]` 冻结字段（脚本任务可直接用）：

```json
{
  "id": "uuid",
  "batchId": "uuid",
  "taskType": "SCRIPT_GENERATION",
  "status": "QUEUED|RUNNING|SUCCEEDED|FAILED",
  "priority": 120,
  "targetType": "batch",
  "targetId": "uuid",
  "requestPayload": {},
  "resultPayload": {},
  "errorCode": null,
  "errorMessage": null,
  "retryCount": 0,
  "maxRetries": 3,
  "queuedAt": "ISO8601|null",
  "startedAt": "ISO8601|null",
  "finishedAt": "ISO8601|null",
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601"
}
```

### 4.1 任务状态机（前端展示建议）

`QUEUED -> RUNNING -> SUCCEEDED`

失败分支：

- 可重试错误：`RUNNING -> QUEUED`（`retryCount` 增加）
- 不可重试错误：`RUNNING -> FAILED`

当前不可重试典型错误码：

- `SCRIPT_PROVIDER_AUTH`
- `SCRIPT_PROVIDER_INVALID_RESPONSE`
- `SCRIPT_PROVIDER_BAD_REQUEST`
- `SCRIPT_PROVIDER_CONFIG`

可重试典型错误码：

- `SCRIPT_PROVIDER_TIMEOUT`
- `SCRIPT_PROVIDER_NETWORK`
- `SCRIPT_PROVIDER_RATE_LIMIT`
- `SCRIPT_PROVIDER_UPSTREAM`

其他内部错误（例如批次前置条件缺失）可能仍走重试，错误码为 `WORKER_ERROR`。

## 7. 统一错误结构

所有接口失败统一返回：

```json
{
  "error": {
    "message": "具体错误信息"
  }
}
```

## 8. 前端 fetch 最小联调代码

```js
const BASE_URL = "http://localhost:3000";
const BATCH_ID = "00000000-0000-0000-0000-000000000001";
const REFERENCE_VIDEO_ID = "00000000-0000-0000-0000-000000000101";

async function request(path, options = {}) {
  const res = await fetch(`${BASE_URL}${path}`, {
    headers: {
      "Content-Type": "application/json",
      ...(options.headers || {}),
    },
    ...options,
  });
  const body = await res.json();
  if (!res.ok) throw new Error(body?.error?.message || `HTTP ${res.status}`);
  return body;
}

export async function startScriptGeneration() {
  await request(`/api/batches/${BATCH_ID}`, {
    method: "PATCH",
    body: JSON.stringify({
      status: "scripts_ready",
      scriptTargetCount: 10,
      productProfile: {
        productSummary: "fetch 联调更新",
      },
    }),
  });

  await request(`/api/batches/${BATCH_ID}/reference-videos/select`, {
    method: "POST",
    body: JSON.stringify({
      referenceVideoId: REFERENCE_VIDEO_ID,
      querySnapshot: { keyword: "pet deodorizer", sortBy: "play_count_desc" },
    }),
  });

  return request(`/api/batches/${BATCH_ID}/script-generation-tasks`, {
    method: "POST",
    body: JSON.stringify({
      desiredCount: 10,
      triggerSource: "ui_fetch",
      requestPayload: { operator: "fetch-debug" },
    }),
  });
}

export async function pollBatchDetail() {
  return request(`/api/batches/${BATCH_ID}`);
}
```

## 10. 测试与 Worker

部署与上线流程见：

- `部署与上线最小清单.md`

```bash
npm run test:integration
```

部署前一键校验（含 Prisma/worker 配置检查与集成测试）：

```bash
npm run release:preflight
```

运行 worker 前请先配置：

- `SCRIPT_GENERATION_API_KEY`（通用）或 `OPENAI_API_KEY` 或 `GEMINI_API_KEY`
- 可选：`SCRIPT_GENERATION_PROVIDER`、`SCRIPT_GENERATION_MODEL`、`SCRIPT_GENERATION_BASE_URL`、`SCRIPT_GENERATION_TIMEOUT_MS`

说明：

- worker 启动时会先做运行时配置校验。
- 若 key 缺失或 provider 配置无效，worker 会直接报错退出，且不会 claim 队列任务。

```bash
npm run worker:scripts:once
```

可选限制任务数：

```bash
node src/workers/script-generation-worker.js --limit=5
```

定时调度接入（基于 `worker:scripts:once`）：

- 已提供工作流：`.github/workflows/worker-scripts-scheduler.yml`
- 可在 GitHub Actions 页面手动触发或按 cron 周期触发
