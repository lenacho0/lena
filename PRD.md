# PRD：爆款视频批量拆解与九宫格分镜生成平台 V1.0 最终定稿版

**文档信息**
- 产品名称：爆款视频批量拆解与九宫格分镜生成平台
- 文档版本：V1.0 Final
- 文档日期：2026-03-23
- 产品阶段：MVP / 内部生产提效工具

## 一、产品背景
团队当前需要批量生产宠物用品广告视频，用于测试不同脚本方向的效果，并围绕效果好的脚本进行裂变放量。现有流程依赖人工完成爆款视频分析、脚本撰写、分镜拆解和分镜图生成，效率低、并发差、标准不统一。

V1 的核心目标不是直接生成最终视频，而是批量产出可测试的创意资产：
- 脚本
- 九镜头 JSON
- 九宫格分镜图

这些结果将交由外部视频生成平台继续生产视频并投入测试。

## 二、产品目标
- 支持从 TikTok 爆款视频中筛选并选择 1 条参考视频
- 支持在本工具内完成参考视频素材获取与下载
- 支持通过 Gemini 对视频进行结构化拆解分析
- 支持结合产品资料、产品图、参考角色图批量生成多版脚本
- 支持将脚本拆成 9 镜头结构化 JSON，并允许人工编辑
- 支持基于全局一致性约束一次性生成整张九宫格分镜图
- 支持多任务并发，批量生成多个脚本和多个九宫格
- 支持合规风险提示
- 支持导出脚本文档、九镜头 JSON、九宫格图片

## 三、非目标
- V1 不生成最终视频
- V1 不做投放平台自动回传
- V1 不提供多参考视频的用户侧能力
- V1 不做自动裂变优胜脚本
- V1 不做复杂权限、审批流、团队协作系统

## 四、目标用户
- 电商内容团队
- 广告投放团队
- 创意策划 / 编导
- 宠物用品品牌运营团队

## 五、版本策略
### V1 版本策略：单视频能力，多视频兼容架构
V1 在用户可见能力上采用“单参考视频”模式，即每个 Batch 仅允许用户选择 1 条参考视频。

这样做是为了优先保证：
- 流程简单
- 结果可解释
- 首版稳定性更高
- 更快验证“批量脚本生产 + 九宫格分镜生成”的业务价值

同时，技术架构需按“多视频兼容”预留：
- 数据结构支持参考视频集合
- 分析层支持综合结果对象
- 结果对象保留参考来源追踪能力

后续 V2 可升级为：
- 多参考视频选择
- 多视频分析结果融合
- 权重配置
- 冲突风格消解
- 多视频脚本生成优化

## 六、核心价值
- 将爆款视频灵感转成可批量复用的创意资产
- 将单次创作升级成批量测试
- 将创作经验沉淀成结构化数据
- 为后续视频量产和投放测试提供稳定输入

## 七、核心使用场景
1. 运营通过筛选条件找到目标爆款视频并选作参考
2. 系统拆解参考视频的钩子、节奏、卖点与转化方式
3. 用户上传宠物用品产品资料与角色参考图，批量生成多版脚本
4. 用户筛选合适脚本后，批量生成九宫格分镜图
5. 将结果导出到外部视频平台继续批量生产视频

## 八、产品范围
V1 包含以下能力：
- 爆款视频筛选与选择
- 视频素材下载 / 素材准备
- Gemini 视频结构化分析
- 产品资料与素材上传
- 批量脚本生成
- 九镜头 JSON 自动拆解与编辑
- 九宫格整图生成
- 并发任务执行
- 合规风险提示
- 结果导出

## 九、核心流程
1. 用户创建创意批次 Batch
2. 用户配置筛选条件并通过 FastMoss 查询视频
3. 用户选择 1 条参考视频
4. 系统在工具内完成该视频的素材获取与下载
5. 系统调用 Gemini 输出爆款视频分析 JSON
6. 用户录入产品资料并上传产品图
7. 用户上传真人参考图、宠物参考图
8. 用户发起批量脚本生成任务
9. 系统默认生成 10 条脚本
10. 用户筛选脚本
11. 系统将被选中的脚本拆成九镜头 JSON
12. 用户可编辑全部 JSON 字段
13. 系统为所选脚本生成整张九宫格分镜图
14. 用户导出脚本、JSON、九宫格结果

## 十、功能需求
### 10.1 批次管理
- 支持创建、查看、管理 Batch
- 展示 Batch 进度、产物数量、失败数量
- 同一 Batch 下管理参考视频、产品资料、脚本和九宫格结果

### 10.2 爆款视频检索
- 接入 FastMoss 官方 API
- 支持用户自定义筛选条件
- 支持按播放量、点赞量、互动率、发布时间等排序
- 支持展示视频基础信息、封面、作者、核心指标
- 支持从结果中选择 1 条参考视频

### 10.3 视频下载与素材准备
- V1 必须在工具内实现参考视频素材获取与下载
- 输出统一的视频分析输入资产
- 若主视频文件获取失败，可定义降级素材方案，但主能力必须可用

### 10.4 视频分析
- 将视频提交给 Gemini
- 输出结构化爆款分析 JSON
- 包含：钩子、结构、节奏、口播、字幕、卖点、情绪曲线、CTA、适配品类等

### 10.5 产品资料管理
- 支持录入产品基础资料
- 支持上传 1 张高质量白底产品图
- 支持上传真人参考角色图
- 支持上传宠物参考角色图
- 支持录入允许表述、禁止表述、必须保留卖点

### 10.6 批量脚本生成
- 基于 1 条参考视频 + 1 个产品资料包生成多条脚本
- 默认生成 10 条脚本
- 默认脚本风格基类：
  - 稳妥转化型
  - 强钩子型
  - 氛围种草型
- 单批次最多 50 条脚本
- 每条脚本需附带生成来源和标签

### 10.7 九镜头 JSON 拆解
- 将脚本拆解为 9 个标准镜头卡
- JSON 中必须包含全局一致性约束
- 每个镜头至少包含：
  - 画面描述
  - 机位
  - 景别
  - 动作
  - 产品位置
  - 时长建议
- 用户可编辑全部字段
- 支持恢复系统默认版本

### 10.8 九宫格分镜生成
- 九宫格必须基于“全局约束 + 9 镜头卡”一次性生成整张图
- 不采用 9 张图独立生成
- 每条脚本默认生成 1 张九宫格
- 单脚本最多生成 3 张九宫格
- 目标保证：
  - 真人角色一致
  - 宠物角色一致
  - 产品一致
  - 场景与风格一致

### 10.9 分镜图画面约束
- 九宫格分镜图中不允许出现任何字幕、贴纸、角标、按钮、Logo 文案、水印风叠加元素
- 所有镜头信息仅存在于 JSON 和系统编辑界面中
- 分镜图只作为纯视觉参考，不叠加文本层

### 10.10 合规能力
- 对脚本和分镜结果进行合规检查
- 检查重点：
  - 功效夸大
  - 医疗暗示
  - 未授权卖点
  - 与产品图不一致
- V1 仅做风险提示，不做阻断
- 需输出风险等级、问题说明、建议

### 10.11 导出能力
- 支持按 Batch 导出
- 导出仅包含：
  - 脚本文档
  - 九镜头 JSON
  - 九宫格图片

## 十一、并发与任务系统要求
- 所有生成环节必须异步化
- 支持多任务并发
- 支持脚本任务与九宫格任务分开调度
- 任务状态至少包括：
  - `draft`
  - `queued`
  - `running`
  - `succeeded`
  - `failed`
  - `partial_success`
- 支持失败重试
- 支持批次级进度统计
- 支持控制并发上限，避免资源拥堵和成本失控

## 十二、关键业务规则
- 每个 Batch 在数据结构上支持参考视频集合
- V1 产品规则限制为每个 Batch 最多选择 1 条参考视频
- 每个 Batch 绑定 1 个产品资料包
- 产品图 V1 固定为 1 张高质量白底图
- 真人参考图、宠物参考图为可上传输入
- 若未上传角色参考图，系统需提示一致性风险
- 默认先生成脚本，用户筛选后再生成九宫格
- 合规风险只提示，不阻断
- 视频下载功能属于 V1 必做范围
- 九宫格生成结果不得出现文字叠加元素

## 十五、成功指标
- 单批次可稳定生成 10 条脚本
- 九宫格生成成功率不低于 80%
- 同批次内角色和产品一致性达到可接受水平
- 导出的脚本和九宫格可直接交给外部视频平台继续生产
- 视频下载功能在工具内可正常使用
- 九宫格分镜图中不出现字幕、贴纸、角标等覆盖元素

## 十六、风险与待补方案
- 视频下载链路的稳定性和可用性需单独设计实现方案
- 角色参考图质量会直接影响一致性效果
- 并发批量生成会放大模型调用成本
- 宠物用品类目的合规规则仍需后续细化
- 九宫格整图生成的稳定性需通过实际样本持续调优

## 十七、MVP 验收标准
- 能创建 Batch 并选中 1 条参考视频
- 能在工具内完成参考视频素材获取与下载
- 能完成参考视频结构化分析
- 能录入产品资料并上传产品图、角色图
- 能批量生成脚本
- 能将脚本拆为可编辑的 9 镜头 JSON
- 能一次性生成整张九宫格
- 能并发执行多个脚本和多个九宫格任务
- 能导出脚本、JSON、九宫格图片
- 能输出基础合规风险提示

## 十八、后续版本方向
- 多参考视频用户侧能力
- 多视频融合与权重配置
- 九宫格直接转视频
- 投放结果回写与标签闭环
- 优胜脚本裂变
